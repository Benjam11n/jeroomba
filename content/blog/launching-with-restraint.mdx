The easiest mistake in a personal site is treating it like a product launch.

That usually leads to too many surfaces, too much animation, and too many
features that do nothing for the reader. A stronger default is simpler:

1. Ship a clear homepage.
2. Add a writing surface.
3. Publish a few case studies that explain what you did and why it mattered.

For a portfolio, **clarity compounds faster than novelty**. If someone lands on
the site and understands your work in under a minute, the design has done its
job.

## What this starter optimizes for

- Fast rendering with server components by default.
- Blog posts stored in the repo as MDX.
- A visual system that feels intentional without getting in the way.

```tsx
export function Principle() {
  return <p>Make the work easier to understand.</p>;
}
```

That tradeoff is especially useful when the site has to live for years instead
of one launch cycle.
